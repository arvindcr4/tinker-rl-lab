  <h1>
    <a href="#claim-support-experiment-matrix" name="claim-support-experiment-matrix" class="pilcrow"></a>
Claim-Support Experiment Matrix
  </h1>
</div>
<p>This matrix is the minimum credible package for defending a claim stronger than:</p>
<blockquote>
<p>&quot;We optimized GSM8K.&quot;</p>
</blockquote>
<p>It is designed to answer the reviewer objection that GSM8K saturation alone does not establish robust mathematical reasoning.</p>
<div class="pilwrap" id="what-this-package-tests">
  <h2>
    <a href="#what-this-package-tests" name="what-this-package-tests" class="pilcrow"></a>
What This Package Tests
  </h2>
</div>
<ol>
<li>
<p><code>Matched-family scaling</code>
Why: separates a real scaling trend from a one-off 3B vs 8B anecdote.</p>
</li>
<li>
<p><code>Replication</code>
Why: shows the result is not a lucky single trajectory.</p>
</li>
<li>
<p><code>Prompt-prefix ablation</code>
Why: checks whether the worked-example prefix is carrying the result.</p>
</li>
<li>
<p><code>Bootstrap-threshold control</code>
Why: distinguishes model capacity from initial-signal / task-difficulty effects.</p>
</li>
<li>
<p><code>Harder in-domain math</code>
Why: tests whether GSM8K gains survive on a more difficult symbolic-math benchmark.</p>
</li>
<li>
<p><code>Out-of-distribution benchmark evaluation</code>
Why: tests whether the trained checkpoint generalizes beyond one saturated benchmark.</p>
</li>
</ol>

  <h2>
    <a href="#out-of-distribution-evaluation" name="out-of-distribution-evaluation" class="pilcrow"></a>
Out-of-Distribution Evaluation
  </h2>
</div>
<p>Use <a href="/home/arvind/tinker-rl-lab/atropos/eval_reasoning_suite.py.html">eval_reasoning_suite.py</a> to evaluate a base model or saved checkpoint on:</p>
<ul>
<li><code>gsm8k</code></li>
<li><code>gsm1k</code></li>
<li><code>gsm_symbolic_main</code></li>
<li><code>gsm_symbolic_p1</code></li>
<li><code>gsm_symbolic_p2</code></li>
<li><code>math</code></li>
<li><code>olympiadbench</code></li>
</ul>
<p>The intended use is:</p>
<ol>
<li>Train a checkpoint on GSM8K.</li>
<li>Serve that checkpoint with <a href="/home/arvind/tinker-rl-lab/atropos/serve.py.html">serve.py</a>.</li>
<li>Run deterministic eval on multiple benchmarks.</li>
</ol>

How These Results Support Claims
  </h2>
</div>
<p>You can defend progressively stronger statements depending on what the results show.</p>
<div class="pilwrap" id="weak-but-safe">
  <h3>
    <a href="#weak-but-safe" name="weak-but-safe" class="pilcrow"></a>
Weak but safe
  </h3>
</div>
<ul>
<li>&quot;GRPO + LoRA strongly optimizes GSM8K under deterministic exact-match evaluation.&quot;</li>
</ul>
<div class="pilwrap" id="medium">
  <h3>
    <a href="#medium" name="medium" class="pilcrow"></a>
Medium
  </h3>
</div>
<p>Requirements:</p>
<ul>
<li>multi-seed GSM8K stability</li>
<li>no-prefix still improves materially</li>
<li>MATH also improves</li>
</ul>
<p>Claim:</p>
<ul>
<li>&quot;The gains are not reducible to a single seed or a fixed worked-example prefix, and transfer to a harder symbolic-math benchmark.&quot;</li>
</ul>
<div class="pilwrap" id="stronger">
  <h3>
    <a href="#stronger" name="stronger" class="pilcrow"></a>
Stronger
  </h3>
</div>
<p>Requirements:</p>
<ul>
<li>GSM8K-trained checkpoint improves on <code>gsm1k</code> or <code>gsm_symbolic_*</code></li>
<li>GSM8K-trained checkpoint also improves on <code>math</code> and/or <code>olympiadbench</code></li>
</ul>
<p>Claim:</p>
<ul>
<li>&quot;The post-training gains extend beyond within-benchmark GSM8K optimization and show evidence of broader mathematical generalization.&quot;</li>
</ul>
<div class="pilwrap" id="what-not-to-claim-even-after-running-this">
  <h2>
    <a href="#what-not-to-claim-even-after-running-this" name="what-not-to-claim-even-after-running-this" class="pilcrow"></a>
What Not To Claim Even After Running This
  </h2>
</div>
<p>Still avoid:</p>
<ul>
<li>&quot;robust mathematical reasoning&quot; as a blanket claim from GSM8K alone</li>
<li>&quot;agentic RL&quot; from these experiments</li>
<li>&quot;pure RL with no supervision&quot; if the prompt prefix is enabled</li>
<li>&quot;capacity threshold&quot; unless the bootstrap controls and intermediate scale points agree</li>
</ul>
