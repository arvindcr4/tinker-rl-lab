  <h1>
    <a href="#tinker-atropos" name="tinker-atropos" class="pilcrow"></a>
tinker-atropos
  </h1>
</div>
<p>An integration layer connecting Atropos (https://github.com/NousResearch/atropos) with the Thinking Machines Tinker API (https://thinkingmachines.ai/tinker/). This package enables seamless model training with Atropos environments from your local machine, abstracting away compute management and infrastructure concerns.</p>
<div class="pilwrap" id="installation">
  <h2>
    <a href="#installation" name="installation" class="pilcrow"></a>
Installation
  </h2>
</div>
<pre><code class="bash">pip install -e .
</code></pre>
<div class="pilwrap" id="quickstart">
  <h2>
    <a href="#quickstart" name="quickstart" class="pilcrow"></a>
Quickstart
  </h2>
</div>
<p>First, obtain a Tinker API key from https://tinker-console.thinkingmachines.ai/keys.</p>
<p>Run the following commands in separate terminal windows to start a training run:</p>
<pre><code class="bash"><span class="hljs-comment"># Terminal 1: Start Atropos API</span>
run-api

<span class="hljs-comment"># Terminal 2: Start training</span>
<span class="hljs-built_in">export</span> TINKER_API_KEY=<span class="hljs-string">"&lt;your-key&gt;"</span>
python launch_training.py --config configs/default.yaml

<span class="hljs-comment"># Terminal 3: Start environment (use built-in or any Atropos environment)</span>
python tinker_atropos/environments/gsm8k_tinker.py serve --config configs/default.yaml
</code></pre>
<p>This runs a 50-step training example with Llama-3.1-8B-Instruct on the GSM8k environment.</p>
<div class="pilwrap" id="using-any-atropos-environment">
  <h2>
    <a href="#using-any-atropos-environment" name="using-any-atropos-environment" class="pilcrow"></a>
Using Any Atropos Environment
  </h2>
</div>
<p><strong>You can use any existing Atropos environment directly with Tinker!</strong> Just point to it and pass your Tinker config:</p>
<pre><code class="bash"><span class="hljs-comment"># Use any Atropos environment with Tinker training</span>
python /path/to/atropos/environment.py serve --config /path/to/your_tinker_config.yaml
</code></pre>
<div class="pilwrap" id="example-with-atropos-math-environment">
  <h3>
    <a href="#example-with-atropos-math-environment" name="example-with-atropos-math-environment" class="pilcrow"></a>
Example with Atropos Math Environment
  </h3>
</div>
<pre><code class="bash"><span class="hljs-comment"># Terminal 1: Start Atropos API</span>
run-api

<span class="hljs-comment"># Terminal 2: Start Tinker training with math config</span>
<span class="hljs-built_in">export</span> TINKER_API_KEY=<span class="hljs-string">"&lt;your-key&gt;"</span>

<span class="hljs-comment"># Terminal 3: Use Atropos math environment with Tinker config</span>
python ~/atropos/environments/math_server.py serve --config configs/math_config.yaml
</code></pre>
<div class="pilwrap" id="how-it-works">
  <h3>
    <a href="#how-it-works" name="how-it-works" class="pilcrow"></a>
How It Works
  </h3>
</div>
<p>Atropos environments support a <code>--config</code> flag that loads your Tinker config (which follows the standard Atropos format with a <code>tinker</code> section). The environment uses:</p>
<ul>
<li>The <code>env</code> section for environment configuration</li>
<li>The <code>openai</code> section for inference server configuration</li>
<li>The <code>tinker</code> section is used by the trainer (ignored by the environment)</li>
</ul>
<p><strong>No modifications needed</strong> - any Atropos environment using <code>managed_server</code> is compatible!</p>
<div class="pilwrap" id="built-in-example-environments">
  <h3>
    <a href="#built-in-example-environments" name="built-in-example-environments" class="pilcrow"></a>
Built-in Example Environments
  </h3>
</div>
<p>This repo includes an example environment in <code>tinker_atropos/environments/</code>:</p>
<ul>
<li><code>gsm8k_tinker.py</code> - Math reasoning with GSM8k dataset</li>
</ul>
<p>These demonstrate integration patterns and custom reward functions.</p>
<div class="pilwrap" id="downloading-weights">
  <h2>
    <a href="#downloading-weights" name="downloading-weights" class="pilcrow"></a>
Downloading Weights
  </h2>
</div>
<p>The trainer outputs a Tinker path at the end of training:</p>
<pre><code>tinker://&lt;training_run_id&gt;/sampler_weights/final
</code></pre>
<p>where <code>&lt;training_run_id&gt;</code> is the Training Run ID from https://tinker-console.thinkingmachines.ai.</p>
<p>To download the weights, set the <code>TINKER_PATH</code> in <code>tinker_atropos/utils/download_weights.py</code> and run:</p>
<pre><code class="bash">python tinker_atropos/utils/download_weights.py
</code></pre>
<p>Weights will be saved to the specified location.</p>
<div class="pilwrap" id="configuration">
  <h2>
    <a href="#configuration" name="configuration" class="pilcrow"></a>
Configuration
  </h2>
</div>
<p>Both the trainer and environment support YAML configuration files to ensure parameter consistency across your training pipeline. The provided environment file (<code>gsm8k_tinker.py</code>) references a configuration path that should match the trainer's configuration. Update the <code>CONFIG_PATH</code> variable in the environment file when switching between configurations.</p>
<div class="pilwrap" id="usage">
  <h3>
    <a href="#usage" name="usage" class="pilcrow"></a>
Usage
  </h3>
</div>
<pre><code class="bash"><span class="hljs-comment"># Use default configuration</span>
python launch_training.py

<span class="hljs-comment"># Use a specific config file</span>
python launch_training.py --config configs/quick_test.yaml

<span class="hljs-comment"># Override parameters via CLI</span>

</code></pre>
<div class="pilwrap" id="available-configs">
  <h3>
    <a href="#available-configs" name="available-configs" class="pilcrow"></a>
Available Configs
  </h3>
</div>
<ul>
<li><code>default.yaml</code> - Standard GSM8k config (50 steps, batch_size=128, Llama-3.1-8B)</li>
<li><code>quick_test.yaml</code> - Quick test (10 steps, Llama-3.1-8B, no wandb)</li>
</ul>
<div class="pilwrap" id="configuration-structure">
  <h3>
    <a href="#configuration-structure" name="configuration-structure" class="pilcrow"></a>
Configuration Structure
  </h3>
</div>
<p>Configs follow the Atropos format with a <code>tinker</code> section for Tinker-specific settings:</p>
<pre><code class="yaml"><span class="hljs-attr">env:</span>
  <span class="hljs-comment"># Standard Atropos environment config</span>
  <span class="hljs-attr">group_size:</span> <span class="hljs-number">16</span>
  <span class="hljs-attr">batch_size:</span> <span class="hljs-number">128</span>
  <span class="hljs-attr">tokenizer_name:</span> <span class="hljs-string">"meta-llama/Llama-3.1-8B-Instruct"</span>
  <span class="hljs-comment"># ... more settings</span>

<span class="hljs-attr">openai:</span>
  <span class="hljs-comment"># Standard Atropos server config</span>
  <span class="hljs-bullet">-</span> <span class="hljs-attr">model_name:</span> <span class="hljs-string">"meta-llama/Llama-3.1-8B-Instruct"</span>
    <span class="hljs-attr">base_url:</span> <span class="hljs-string">"http://localhost:8001/v1"</span>
    <span class="hljs-comment"># ... more settings</span>

<span class="hljs-attr">tinker:</span>
  <span class="hljs-comment"># Tinker-specific training config</span>
  <span class="hljs-attr">lora_rank:</span> <span class="hljs-number">32</span>
  <span class="hljs-attr">learning_rate:</span> <span class="hljs-number">0.00004</span>
  <span class="hljs-comment"># ... more settings</span>
</code></pre>
<p>See <code>configs/default.yaml</code> for the complete configuration options.</p>
<div class="pilwrap" id="adapting-existing-atropos-configs">
  <h3>
    <a href="#adapting-existing-atropos-configs" name="adapting-existing-atropos-configs" class="pilcrow"></a>
Adapting Existing Atropos Configs
  </h3>
</div>
<p>To use an existing Atropos environment config with Tinker:</p>
<ol>
<li>
<p>Add a <code>tinker</code> section with training parameters:</p>
<pre><code class="yaml"><span class="hljs-attr">tinker:</span>
  <span class="hljs-attr">lora_rank:</span> <span class="hljs-number">32</span>
  <span class="hljs-attr">learning_rate:</span> <span class="hljs-number">0.00004</span>
  <span class="hljs-attr">max_token_trainer_length:</span> <span class="hljs-number">2048</span>
  <span class="hljs-attr">checkpoint_dir:</span> <span class="hljs-string">"./checkpoints/"</span>
  <span class="hljs-attr">save_checkpoint_interval:</span> <span class="hljs-number">0</span>
  <span class="hljs-attr">wandb_project:</span> <span class="hljs-string">"my-project"</span>
  <span class="hljs-attr">wandb_group:</span> <span class="hljs-literal">null</span>
  <span class="hljs-attr">wandb_run_name:</span> <span class="hljs-string">"my-run"</span>
</code></pre>
</li>
<li>
<p>Ensure your environment uses <code>managed_server</code> with stop sequences:</p>
<pre><code class="python"><span class="hljs-keyword">async</span> <span class="hljs-keyword">with</span> self.server.managed_server(tokenizer=self.tokenizer) <span class="hljs-keyword">as</span> managed:
    chat_completion = <span class="hljs-keyword">await</span> managed.chat_completion(
        messages=messages,
        n=self.config.group_size,
        max_tokens=self.config.max_token_length,
        temperature=<span class="hljs-number">1.0</span>,
        stop=[self.tokenizer.eos_token_id],  <span class="hljs-comment"># Add stop sequences</span>
    )
</code></pre>
</li>
<li>
<p>That's it! The config is ready to use with both Atropos and Tinker.</p>
</li>
</ol>
