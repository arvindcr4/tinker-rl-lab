    <div class="docs markdown"><div class="pilwrap" id="tinker-atropos-configuration-files">
  <h1>
    <a href="#tinker-atropos-configuration-files" name="tinker-atropos-configuration-files" class="pilcrow"></a>
Tinker-Atropos Configuration Files
  </h1>
</div>
<p>This directory contains YAML configuration files for the Tinker-Atropos training system. Config files provide an easy way to manage different training environments and can be combined with CLI overrides.</p>
<div class="pilwrap" id="available-configurations">
  <h2>
    <a href="#available-configurations" name="available-configurations" class="pilcrow"></a>
Available Configurations
  </h2>
</div>
<div class="pilwrap" id="default.yaml">
  <h3>
    <a href="#default.yaml" name="default.yaml" class="pilcrow"></a>
<code>default.yaml</code>
  </h3>
</div>
<p>Standard configuration suitable for typical training runs. This config demonstrates all available options with reasonable defaults.</p>
<p><strong>Use case</strong>: Regular training experiments with moderate resources</p>
<div class="pilwrap" id="quick_test.yaml">
  <h3>
    <a href="#quick_test.yaml" name="quick_test.yaml" class="pilcrow"></a>
<code>quick_test.yaml</code>
  </h3>
</div>
<p>Minimal resource configuration optimized for rapid testing and debugging.</p>
<p><strong>Features</strong>:</p>
<ul>
<li>Reduced batch size and token lengths</li>
<li>Fewer workers and shorter training steps</li>
<li>Wandb disabled by default</li>
<li>Ideal for validating setup and code changes</li>
</ul>
<p><strong>Use case</strong>: Local testing, debugging, CI/CD pipelines</p>
<div class="pilwrap" id="configuration-parameters">
  <h2>
    <a href="#configuration-parameters" name="configuration-parameters" class="pilcrow"></a>
Configuration Parameters
  </h2>
</div>
<div class="pilwrap" id="tinker-model-configuration">
  <h3>
    <a href="#tinker-model-configuration" name="tinker-model-configuration" class="pilcrow"></a>
Tinker Model Configuration
  </h3>
</div>
<ul>
<li><code>base_model</code>: HuggingFace model identifier (default: &quot;meta-llama/Llama-3.1-8B-Instruct&quot;)</li>
<li><code>lora_rank</code>: Rank of LoRA adapters (default: 32)</li>
<li><code>learning_rate</code>: Optimizer learning rate (default: 0.00004)</li>
</ul>
<div class="pilwrap" id="training-configuration">
  <h3>
    <a href="#training-configuration" name="training-configuration" class="pilcrow"></a>
Training Configuration
  </h3>
</div>
<ul>
<li><code>num_steps</code>: Total number of training steps (default: 100)</li>
<li><code>batch_size</code>: Number of datums per training batch (default: 128)</li>
<li><code>group_size</code>: Number of rollouts generated per question (default: 16)</li>
<li><code>max_token_env_length</code>: Maximum tokens for environment rollouts (default: 256)</li>
<li><code>max_token_trainer_length</code>: Maximum tokens for trainer processing (default: 2048)</li>
<li><code>max_num_workers</code>: Maximum parallel workers for rollout generation (calculated as (batch_size * num_offpolicy // group_size)) (default: 24)</li>
<li><code>max_batches_offpolicy</code>: Maximum batches before data considered stale (default: 3)</li>
</ul>
<div class="pilwrap" id="weights-and-biases-configuration">
  <h3>
    <a href="#weights-and-biases-configuration" name="weights-and-biases-configuration" class="pilcrow"></a>
Weights &amp; Biases Configuration
  </h3>
</div>
<ul>
<li><code>use_wandb</code>: Enable/disable wandb logging (default: true)</li>
<li><code>wandb_project</code>: Wandb project name (default: &quot;atropos-tinker&quot;)</li>
<li><code>wandb_group</code>: Wandb group name for organizing runs (default: null, auto-generated)</li>
<li><code>wandb_run_name</code>: Base name for wandb runs (default: &quot;atropos-tinker-run&quot;)</li>
<li><code>wandb_run_suffix</code>: Random suffix for unique run names (auto-generated)</li>
</ul>
<div class="pilwrap" id="api-endpoints">
  <h3>
    <a href="#api-endpoints" name="api-endpoints" class="pilcrow"></a>
API Endpoints
  </h3>
</div>
<ul>
<li><code>atropos_api_url</code>: URL for Atropos rollout server (default: &quot;http://localhost:8000&quot;)</li>
<li><code>inference_api_url</code>: URL for unified trainer inference endpoint (default: &quot;http://localhost:8001&quot;)</li>
</ul>
<div class="pilwrap" id="evaluation">
  <h3>
    <a href="#evaluation" name="evaluation" class="pilcrow"></a>
Evaluation
  </h3>
</div>
<ul>
<li><code>steps_per_eval</code>: Steps between evaluation runs (default: 100)</li>
<li><code>num_requests_for_eval</code>: Number of requests per evaluation (default: 256)</li>
</ul>
<div class="pilwrap" id="debug-options">
  <h3>
    <a href="#debug-options" name="debug-options" class="pilcrow"></a>
Debug Options
  </h3>
</div>
<ul>
<li><code>ensure_scores_not_the_same</code>: Validate that scores differ within groups (default: false)</li>
</ul>

<div class="pilwrap" id="with-cli-overrides">
  <h3>
    <a href="#with-cli-overrides" name="with-cli-overrides" class="pilcrow"></a>
With CLI Overrides
  </h3>
</div>
<pre><code class="bash"><span class="hljs-comment"># Override specific parameters</span>
python launch_training.py --config configs/default.yaml \
    --num-steps 50 \
    --batch-size 64 \
    --no-wandb

<span class="hljs-comment"># Change model and adjust hyperparameters</span>
python launch_training.py --config configs/quick_test.yaml \
    --base-model <span class="hljs-string">"meta-llama/Llama-3.2-1B-Instruct"</span> \
    --learning-rate 0.0001
</code></pre>

<div class="pilwrap" id="creating-custom-configs">
  <h2>
    <a href="#creating-custom-configs" name="creating-custom-configs" class="pilcrow"></a>
Creating Custom Configs
  </h2>
</div>
<p>To create a new configuration:</p>
<ol>
<li>Copy an existing config file as a template</li>
<li>Modify parameters as needed</li>
<li>Save with a descriptive name (e.g., <code>configs/my_experiment.yaml</code>)</li>
<li>Run with: <code>python launch_training.py --config configs/my_experiment.yaml</code></li>
</ol>

<div class="pilwrap" id="configuration-priority">
  <h2>
    <a href="#configuration-priority" name="configuration-priority" class="pilcrow"></a>
Configuration Priority
  </h2>
</div>
<p>When parameters are specified in multiple places, the priority order is:</p>
<ol>
<li><strong>CLI arguments</strong> (highest priority)</li>
<li><strong>YAML config file</strong></li>
<li><strong>Environment variables</strong> (with <code>TINKER_ATROPOS_</code> prefix)</li>
<li><strong>Default values</strong> (lowest priority)</li>
</ol>
<p>Example:</p>
<pre><code class="bash"><span class="hljs-comment"># YAML has num_steps: 100</span>
<span class="hljs-comment"># CLI overrides to 50</span>
python launch_training.py --config configs/default.yaml --num-steps 50
<span class="hljs-comment"># Result: num_steps = 50</span>
</code></pre>
<div class="pilwrap" id="environment-variables">
  <h2>
    <a href="#environment-variables" name="environment-variables" class="pilcrow"></a>
Environment Variables
  </h2>
</div>
<p>All config parameters can be set via environment variables with the <code>TINKER_ATROPOS_</code> prefix:</p>
<pre><code class="bash"><span class="hljs-built_in">export</span> TINKER_ATROPOS_BASE_MODEL=<span class="hljs-string">"meta-llama/Llama-3.2-1B-Instruct"</span>
<span class="hljs-built_in">export</span> TINKER_ATROPOS_NUM_STEPS=200
<span class="hljs-built_in">export</span> TINKER_ATROPOS_USE_WANDB=<span class="hljs-literal">true</span>

python launch_training.py
</code></pre>
<div class="pilwrap" id="tips">
  <h2>
    <a href="#tips" name="tips" class="pilcrow"></a>
Tips
  </h2>
</div>
<ul>
<li>Start with <code>quick_test.yaml</code> to validate your setup</li>
<li>Use <code>default.yaml</code> as a template for custom configs</li>
<li>Enable wandb logging for runs to track metrics</li>
<li>Adjust <code>max_num_workers</code> based on your batch size, group size, and max_batches_offpolicy value</li>
<li>Use CLI overrides for one-off experiments without creating new config files</li>
</ul>
