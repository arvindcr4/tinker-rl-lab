<div class="pilwrap" id="key-observation">
  <h3>
    <a href="#key-observation" name="key-observation" class="pilcrow"></a>
Key Observation
  </h3>
</div>
<p>SOTA models (DeepScaleR, DeepSeek-R1-Distill) start from a <strong>distilled checkpoint</strong> from a 671B teacher model.
Our approach: <strong>pure GRPO from a general-purpose base model</strong> — no distillation, no massive teacher required.</p>

<div class="pilwrap" id="gap-in-literature">
  <h3>
    <a href="#gap-in-literature" name="gap-in-literature" class="pilcrow"></a>
Gap in Literature
  </h3>
</div>
<ol>
<li>Most GRPO results are on <strong>Qwen-family</strong> models only — Llama behavior under GRPO is underexplored</li>
<li>No systematic <strong>scaling study</strong> comparing GRPO across model families (Llama vs Qwen) at multiple sizes</li>
<li>Existing work requires <strong>expensive infrastructure</strong> — our Tinker-based approach is cloud-accessible</li>
<li><strong>Reproducibility</strong>: Most papers lack runnable notebooks</li>
</ol>
<div class="pilwrap" id="our-contribution">
  <h3>
    <a href="#our-contribution" name="our-contribution" class="pilcrow"></a>
Our Contribution
  </h3>
</div>
<blockquote>
<p><strong>Systematic GRPO scaling analysis across model families and sizes (1B→8B→30B) on math reasoning, with reproducible cloud-based experiments on the Tinker platform</strong></p>
</blockquote>

<div class="pilwrap" id="architecture-tinker-+-atropos">
  <h3>
    <a href="#architecture-tinker-+-atropos" name="architecture-tinker-+-atropos" class="pilcrow"></a>
Architecture: Tinker + Atropos
  </h3>
</div>
<pre><code>┌─────────────────┐     ┌──────────────────┐     ┌────────────────────┐
│  GSM8K Env       │────▶│  Atropos API      │────▶│  Tinker Trainer     │
│  (rollouts +     │     │  (coordination)   │     │  (cloud GPU)       │
│   scoring)       │◀────│                    │◀────│  LoRA + GRPO       │
└─────────────────┘     └──────────────────┘     └────────────────────┘
      ▲                                                    │
      │              ┌──────────────────┐                  │
      └──────────────│  Tinker Inference │◀─────────────────┘
                     │  (sampling)       │  updated weights
                     └──────────────────┘  each step
</code></pre>
<ul>
<li><strong>GRPO</strong>: No critic/reward model needed (1x memory vs PPO's 4x)</li>
<li><strong>LoRA</strong>: Rank 32, ~0.1% of parameters trained</li>
<li><strong>Cloud API</strong>: All GPU compute via Tinker — runs from CPU-only laptop</li>
<li><strong>Verifiable rewards</strong>: Binary (correct/incorrect) per GSM8K answer</li>
</ul>
