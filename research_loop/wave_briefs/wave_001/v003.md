# Agent Brief — v003

**Read this brief in full. Then read `research_loop/learnings.md`. Then proceed.**

You are part of a 1000-experiment parallel-agent autoresearch loop for
`tinker-rl-lab`. Your job is to test **one specific hypothesis** against the
current best GRPO recipe and report back a score.

## Challenge

The paper (`reports/final/grpo_agentic_llm_paper.tex`) claims a capacity threshold
between 3B and 4B for GSM8K math reasoning. We want to rescue the 3B failure —
or prove the threshold is real through 1000+ experiments.

Phase 1 uses a fast proxy: Qwen3-0.6B / Qwen3-1.7B, 100 steps, GSM8K 500-example
subset, single seed. Winners are re-evaluated in Phase 2 on the actual 3B models
across 5 seeds.

## Current best recipe

See `research_loop/best_recipe.yaml`. Do not modify it — that's the coordinator's
job.

Current best score: **None** (last10_avg_accuracy).
Your target: **beat None by at least 10%** (a weaker improvement
is discarded as noise).

## Your hypothesis

**ID:** H03
**One-liner:** `group_size=128` is the ceiling before diminishing returns
**Knobs to change:** group_size: 128
**Rationale:** Ryan-Li-style limit test

## Forbidden

- Do not vary knobs outside `group_size: 128`. Stick to the single hypothesis.
  Confounding defeats the purpose of the parallel loop.
- Do not read other agents' briefs or variant configs. Each agent runs
  independently.
- Do not run more than `1` training runs. Fail fast if your first run
  is clearly worse than the current best.
- Do not touch `best_recipe.yaml`, `learnings.md`, `results.jsonl`, or
  `queue.jsonl`. The coordinator owns those.
- Do not spawn sub-agents or sub-processes beyond `run_one.py`.

## What to do

1. Read `research_loop/learnings.md` — skip any dead-end knobs listed there.
2. Read `research_loop/best_recipe.yaml` — copy all fields as your starting
   point.
3. Write your variant to `research_loop/variant_configs/wave_001/v003.yaml`.
   Override ONLY the knobs in `knobs_delta`. Add a `parent: best_recipe_vbaseline_v0`
   field and a `hypothesis: "`group_size=128` is the ceiling before diminishing returns"` field.
4. Run: `python research_loop/run_one.py --config research_loop/variant_configs/wave_001/v003.yaml`
5. The runner will emit `METRIC name=value` lines on stdout. Capture them.
6. Append your result to `research_loop/wave_briefs/wave_001/v003.result.json`
   with this schema:
   ```json
   {
     "variant_id": "v003",
     "hypothesis_id": "H03",
     "parent": "baseline_v0",
     "config_path": "research_loop/variant_configs/wave_001/v003.yaml",
     "metrics": {
       "last10_avg_accuracy": <float>,
       "peak_accuracy": <float>,
       "first5_avg_accuracy": <float>,
       "zero_reward_pct": <float>,
       "zero_loss_pct": <float>
     },
     "delta_vs_baseline": <float>,
     "wall_clock_seconds": <int>,
     "status": "completed" | "failed" | "aborted",
     "notes": "<one sentence: did it help, hurt, or do nothing and why>"
   }
   ```
7. Stop. The coordinator will ingest your result when the wave closes.

## If your run fails

Set `status: failed`, fill `notes` with the error class (OOM, API error,
timeout, spec error), and exit. Do not retry more than once.

## If your run produces a new record

Still only write the result JSON. Do not modify `best_recipe.yaml`. The
coordinator validates winners across 4+ seeds before promoting.
